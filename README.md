# AutoMech — Premium Auto Repair Website Template

A reusable, config-driven website template for local automotive / mechanic shops.
Built with **HTML5, Bootstrap 5, custom CSS and vanilla JavaScript** — no WordPress,
no build framework, no database. Rebrand the entire site for a new client by editing
a **single file**: `config.json`.

> Demo brand in this package: **Deep Auto Care** (Surrey, BC). Replace it with your
> client's details in `config.json` and the whole site updates automatically.

---

## What's included

- **11 page types** — Home, Services, About, Gallery, Reviews, FAQ, Book, Contact,
  Privacy, Terms, 404
- **Auto-generated local-SEO pages** — one landing page per service-area city, each
  with unique title, meta, H1, canonical and schema
- **Config-driven everything** — business info, colours, services, reviews, gallery,
  hours, service areas, social links and SEO all live in `config.json`
- **Lead-generation built in** — floating Call / Text / WhatsApp buttons, sticky
  mobile call bar, appointment + contact forms with validation
- **SEO & schema** — meta tags, Open Graph, Twitter cards, canonical URLs,
  `robots.txt`, `sitemap.xml`, and JSON-LD (AutoRepair / LocalBusiness, Service,
  Review, FAQ, Breadcrumb)
- **Premium design** — glassmorphism, floating cards, soft shadows, scroll-reveal
  animations, fully responsive, accessible (WCAG-minded, keyboard nav, reduced-motion)

## Quick start (60 seconds)

This template loads `config.json` over HTTP using `fetch()`, so it **must be served
by a web server** — opening `index.html` directly with `file://` will not work.

**Option A — Python (already on most machines):**
```bash
cd automech-template
python3 -m http.server 8080
# open http://localhost:8080
```

**Option B — Node:**
```bash
cd automech-template
npx serve .
# or: npx http-server -p 8080
```

Then open the local URL in your browser. That's it — the site renders from
`config.json`.

## Rebrand for a new client

1. Open **`config.json`** and edit the business details, colours, services, reviews,
   gallery, hours, service areas and SEO. (Full field-by-field guide:
   [`docs/CUSTOMIZATION.md`](docs/CUSTOMIZATION.md).)
2. Drop the client's real photos into **`/assets/images`** (same filenames, or update
   the paths in `config.json`). The included images are branded placeholders.
3. Regenerate the local-SEO pages, sitemap and robots file:
   ```bash
   node tools/build.js
   ```
4. Deploy. See [`docs/DEPLOYMENT.md`](docs/DEPLOYMENT.md),
   [`docs/HOSTINGER.md`](docs/HOSTINGER.md) or [`docs/GITHUB.md`](docs/GITHUB.md).

## How it works (architecture)

Every HTML file is a **thin shell**: a `<head>`, three mount points
(`header` / `footer` / `fabs`), an empty `<main data-page-body>`, and a small inline
script that names the page. On load, `js/app.js`:

1. fetches `config.json`,
2. injects the colour theme as CSS variables,
3. renders the page body via `Pages[name](config)`,
4. mounts the shared header / footer / floating buttons,
5. writes SEO meta + the right JSON-LD schema for that page,
6. initialises the form and gallery if present.

Because all content comes from `config.json`, editing that one file rebrands the
entire site — no page-by-page edits.

## Connecting the booking form

By default the form runs in **demo mode** (it validates and shows a success message
but doesn't send anywhere). To receive real submissions, set `booking.endpoint` in
`config.json` to a URL that accepts a JSON `POST`. See
[`docs/MAINTENANCE.md`](docs/MAINTENANCE.md#connecting-the-booking-form) for options
(Formspree, Basin, a serverless function, etc.).
