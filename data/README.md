# Data

`config.json` (in the project root) is the **single source of truth** for the entire
website — business details, services, reviews, gallery, service areas, colours and SEO.

This `/data` folder is reserved for optional future data partials (for example, if a
client wants to split a very large reviews list or blog content into separate JSON
files). By default it is unused — edit the root `config.json` to rebrand the site.
